# 05 - Interview Preparation

# Goal

Become capable of performing strongly in startup engineering interviews.

---

# Interview Categories

## Behavioral

Prepare stories covering:

* Learning software engineering
* Self-teaching journey
* Technical challenges
* Building projects
* Handling failures
* Learning new technologies
* Working independently

Use the STAR framework:

* Situation
* Task
* Action
* Result

---

# Portfolio Walkthrough

For every project explain:

* Problem
* Users
* Architecture
* Stack choices
* Challenges
* Lessons learned
* Future roadmap

Target:

Explain any project clearly within two minutes.

---

# Live Coding

Focus on:

* Problem breakdown
* Communication
* Clean code
* Edge cases
* Debugging

Avoid:

* Silent coding
* Premature optimization
* Guessing

---

# System Design

Practice:

* Requirements gathering
* Component identification
* Database choices
* Caching
* Scaling
* Reliability
* Monitoring

Use diagrams whenever possible.

---

# Mock Interviews

Minimum:

* Two complete mock interviews

Each should include:

* Behavioral
* Portfolio walkthrough
* Live coding
* System design

---

# Startup Research Process

Before interviews:

Research:

* Product
* Customers
* Funding
* Team
* Technology
* Competitors

Prepare thoughtful questions.

---

# Offer Evaluation

Evaluate:

* Compensation
* Equity
* Growth opportunities
* Team quality
* Product quality
* Company stage
* Technical mentorship

---

# Job Readiness Rubric

## Technical

* TypeScript
* SQL
* Backend
* Frontend
* Security
* Testing
* Observability
* AI integration

## Portfolio

* Deployed projects
* Strong GitHub profile
* Clean documentation

## Communication

* Clear explanations
* Architecture reasoning
* Tradeoff discussions

Only consider applications when all three categories are strong.
